02

Ba trụ cột: Commands, Agents, Skills

Gặp gỡ "dàn diễn viên chính" của Claude Code — ba cơ chế mở rộng cốt lõi

Ẩn dụ: Một công ty phần mềm thu nhỏ

Hãy tưởng tượng Claude Code là một công ty phần mềm thu nhỏ. Trong công ty này có ba vai trò chính:

📋
Commands
Giám đốc dự án. Người dùng gọi trực tiếp bằng lệnh /tên-lệnh. Điều phối công việc, hỏi ý kiến người dùng, giao việc cho agents và skills.
🤖
Agents
Chuyên gia kỹ thuật. Làm việc trong "phòng riêng" (context riêng), có bộ công cụ và kiến thức riêng, trả kết quả về cho command.
📚
Skills
Sách chuyên khảo. Kiến thức chuyên sâu có thể được "nạp" vào agent (agent skill) hoặc gọi trực tiếp (inline skill). Tái sử dụng được.

Command: Điểm khởi đầu

Một command nằm trong thư mục .claude/commands/. Đây là file Markdown với phần frontmatter cấu hình:

CODE 
---
description: Fetch weather data for Dubai
model: haiku
---

# Weather Orchestrator Command

### Step 1: Ask User Preference
Use the AskUserQuestion tool...

### Step 2: Fetch Weather Data
Use the Task tool to invoke the
weather agent...

### Step 3: Create SVG Weather Card
Use the Skill tool to invoke the
weather-svg-creator skill...
TIẾNG VIỆT

Phần cấu hình: file này mô tả lệnh lấy dữ liệu thời tiết cho Dubai

Dùng model "haiku" — model nhẹ, nhanh, tiết kiệm — vì command chỉ cần điều phối, không cần suy luận phức tạp

Kết thúc phần cấu hình

Tên lệnh — hiển thị khi gõ /weather-orchestrator

Bước 1: Hỏi người dùng muốn độ C hay F — command tương tác trực tiếp với người dùng

Bước 2: Giao việc cho "chuyên gia" weather-agent — dùng Agent tool để tạo agent riêng biệt

Bước 3: Gọi skill tạo thẻ SVG — dùng Skill tool để nạp kiến thức chuyên biệt vào ngay cuộc hội thoại hiện tại

Agent: Chuyên gia trong phòng riêng

Một agent có context riêng, bộ công cụ riêng, và có thể mang theo preloaded skills:

CODE 
---
name: weather-agent
description: Use this agent PROACTIVELY
  when you need to fetch weather data
model: sonnet
color: green
maxTurns: 5
permissionMode: acceptEdits
memory: project
skills:
  - weather-fetcher
---
TIẾNG VIỆT

Bắt đầu phần cấu hình agent

Tên: "weather-agent" — định danh duy nhất

Từ khoá "PROACTIVELY" rất quan trọng — Claude sẽ TỰ ĐỘNG gọi agent này khi thấy cần lấy dữ liệu thời tiết

Dùng model Sonnet — mạnh hơn Haiku, vì agent cần suy luận phức tạp hơn

Hiển thị màu xanh lá trong terminal — dễ phân biệt

Tối đa 5 lượt hành động — an toàn, không để agent "chạy hoài"

Tự động chấp nhận chỉnh sửa file — không cần hỏi lại người dùng

Có bộ nhớ riêng theo dự án — nhớ lần lấy thời tiết trước

Nạp sẵn skill "weather-fetcher" — kiến thức về API thời tiết được đưa vào ngay khi agent khởi động

Skill: Kiến thức tái sử dụng

Một skill cung cấp kiến thức chuyên biệt — giống như đưa cho Claude một cuốn sách hướng dẫn:

CODE 
---
name: weather-fetcher
description: Instructions for fetching
  weather temperature data
user-invocable: false
---

# Weather Fetcher Skill

1. Fetch Weather Data: Use WebFetch
   URL: api.open-meteo.com/v1/forecast
   ?latitude=25.2048&longitude=55.2708
   ¤t=temperature_2m

2. Extract Temperature:
   Field: current.temperature_2m
TIẾNG VIỆT

Phần cấu hình skill

Tên skill: "weather-fetcher"

Mô tả giúp Claude biết khi nào nên dùng skill này

user-invocable: false — người dùng KHÔNG thể gọi trực tiếp bằng /weather-fetcher. Skill này chỉ được nạp vào agent.

Tên hiển thị của skill

Bước 1: Hướng dẫn cụ thể cách lấy dữ liệu — gọi API Open-Meteo với toạ độ Dubai (25.2°N, 55.3°E)

Bước 2: Chỉ ra chính xác nơi tìm nhiệt độ trong dữ liệu trả về — trường current.temperature_2m

💡
Aha! Hai kiểu Skill — sự khác biệt quan trọng

Một khái niệm then chốt: Agent Skill (nạp sẵn) khác với Inline Skill (gọi trực tiếp). Agent Skill giống sách trong tủ sách của chuyên gia — đã có sẵn khi bắt đầu. Inline Skill giống quyển sách bạn mượn từ thư viện khi cần — gọi bằng /tên-skill trong cuộc hội thoại.

Kiểm tra nhanh

Bạn muốn tạo một workflow mà: (1) hỏi người dùng chọn ngôn ngữ, (2) giao cho AI dịch nội dung, (3) tạo file PDF. Bạn nên bắt đầu bằng thành phần nào?

04

Bộ nhớ và cấu hình

CLAUDE.md, settings.json, và rules — cách "dạy" Claude nhớ về dự án của bạn

Ẩn dụ: Sổ tay nhân viên mới

Khi một nhân viên mới vào công ty, họ nhận được sổ tay hướng dẫn — quy trình, quy tắc, thông tin liên hệ. CLAUDE.md chính là sổ tay đó cho Claude.

📝
CLAUDE.md
Sổ tay dự án. Hướng dẫn cụ thể cho từng dự án — tech stack, quy ước code, cách build và test. Tự động được đọc khi khởi động.
⚙️
settings.json
Nội quy công ty. 60+ cài đặt: quyền hạn, hooks, model AI, MCP servers. Có thể chia sẻ cho cả team.
📏
Rules
Quy tắc theo ngữ cảnh. Áp dụng theo pattern file — ví dụ: chỉ áp dụng cho file *.md hoặc thư mục cụ thể.

settings.json — Trung tâm điều khiển

File settings.json là nơi bạn cấu hình mọi thứ — từ quyền hạn đến hooks:

CODE 
{
  "permissions": {
    "allow": [
      "Edit(*)",
      "WebFetch(domain:api.open-meteo.com)"
    ],
    "deny": [],
    "ask": [
      "Bash(git *)",
      "Bash(rm *)"
    ]
  },
  "outputStyle": "Explanatory",
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "80"
  }
}
TIẾNG VIỆT

Bắt đầu phần cấu hình

Quyền hạn — kiểm soát Claude được làm gì

"allow" = được phép tự động: chỉnh sửa file bất kỳ, gọi API Open-Meteo

Dấu (*) là ký tự đại diện — nghĩa là "tất cả". Edit(*) = được sửa mọi file

"deny" = cấm tuyệt đối (trống = không cấm gì)

"ask" = phải hỏi trước khi làm: các lệnh git và lệnh xoá file — vì đây là hành động nguy hiểm!

Kiểu trả lời: "Giải thích" — Claude sẽ giải thích chi tiết thay vì trả lời ngắn gọn

Biến môi trường: tự động nén context khi đạt 80% — giữ bộ nhớ không bị tràn

Thứ tự ưu tiên cấu hình

Khi nhiều file cấu hình cùng tồn tại, Claude Code dùng thứ tự ưu tiên — giống như luật pháp: luật cao hơn thắng luật thấp hơn:

1

Managed Settings (Tổ chức)

Cài đặt bắt buộc từ tổ chức — không thể ghi đè. Giống quy định pháp luật.

2

Command Line Arguments

Tham số khi khởi chạy Claude — chỉ áp dụng cho phiên hiện tại.

3

.claude/settings.local.json (Cá nhân)

Cài đặt riêng của bạn, git-ignored — không chia sẻ với team.

4

.claude/settings.json (Team)

Cài đặt chung của dự án — commit lên Git, cả team dùng chung.

5

~/.claude/settings.json (Toàn cục)

Cài đặt mặc định cá nhân — áp dụng cho mọi dự án trên máy.

CLAUDE.md trong Monorepo

Trong monorepo, Claude Code có hai cơ chế tải CLAUDE.md:

⬆️

05

Hooks — Hệ thống phản ứng tự động

Dạy Claude Code phản ứng với các sự kiện — từ phát âm thanh đến kiểm tra code

Ẩn dụ: Chuông cửa thông minh

Bạn biết loại chuông cửa thông minh mà tự quay video khi có người đến? Hooks hoạt động y hệt vậy — chúng "lắng nghe" các sự kiện trong Claude Code và tự động phản ứng.

Dự án này dùng hooks để phát âm thanh mỗi khi có sự kiện xảy ra — giúp bạn "nghe" được Claude đang làm gì mà không cần nhìn màn hình.

PreToolUse Trước khi Claude dùng công cụ (đọc file, chạy lệnh...)
PostToolUse Sau khi công cụ hoàn thành — dùng để auto-format code
Stop Khi Claude kết thúc phiên — dùng để kiểm tra cuối cùng
SubagentStart Khi agent con được khởi tạo
SessionStart Khi phiên làm việc mới bắt đầu

Và còn 17 sự kiện khác nữa — tổng cộng 22 hook events!

Cách hooks được cấu hình

Trong settings.json, mỗi hook event được gắn với một hoặc nhiều hành động:

CODE 
"hooks": {
  "PreToolUse": [{
    "hooks": [{
      "type": "command",
      "command": "python3 hooks.py",
      "timeout": 5000,
      "async": true
    }]
  }],
  "Stop": [{
    "hooks": [{
      "type": "command",
      "command": "python3 hooks.py",
      "timeout": 5000,
      "async": true
    }]
  }]
}
TIẾNG VIỆT

Phần cấu hình hooks

Sự kiện "Trước khi dùng công cụ":

Danh sách các hành động sẽ chạy:

Loại: chạy lệnh terminal

Lệnh: chạy script Python xử lý hook — script này nhận sự kiện và phát âm thanh tương ứng

Hết thời gian sau 5 giây — nếu script "đứng" thì bỏ qua

async: true — chạy ngầm, không chặn Claude đang làm việc

Sự kiện "Khi Claude dừng": — cùng script nhưng phát âm thanh khác

Script hooks.py — Bộ não xử lý

File hooks.py là trung tâm xử lý — nó đọc sự kiện và phát âm thanh phù hợp:

CODE 
HOOK_SOUND_MAP = {
  "PreToolUse":    "pretooluse",
  "PostToolUse":   "posttooluse",
  "Stop":          "stop",
  "SubagentStart": "subagentstart",
  "SessionStart":  "sessionstart",
  # ... 22 events total
}
TIẾNG VIỆT

Bảng ánh xạ: mỗi sự kiện → thư mục chứa file âm thanh

Trước khi dùng tool → phát âm từ thư mục "pretooluse/"

Sau khi dùng tool → phát âm từ thư mục "posttooluse/"

Khi dừng → phát âm từ thư mục "stop/"

Khi agent con bắt đầu → phát âm thông báo

Khi phiên mới → phát âm chào mừng

Tổng cộng 22 loại sự kiện, mỗi loại có thư mục âm thanh riêng

💡
Aha! Hooks không chỉ để phát âm thanh

Trong thực tế, hooks mạnh hơn nhiều: auto-format code sau khi edit, kiểm tra TypeScript sau mỗi thay đổi, chặn commit nếu có console.log trong code, cảnh báo khi gọi lệnh nguy hiểm. Bạn đang xây dựng một hệ thống tự bảo vệ cho quá trình phát triển phần mềm.

1

CLAUDE.md dưới 200 dòng

File quá dài → Claude "quên" phần cuối. Chia nhỏ thành nhiều CLAUDE.md ở thư mục con.

2

Dùng Commands thay vì Agents đơn lẻ

Command giữ context chung, có thể điều phối nhiều agent. Agent đơn lẻ bị cô lập.

3

Feature-specific agents với skills

Tạo agent chuyên biệt + nạp skill phù hợp. Không nên tạo agent "đa năng" — nó sẽ không giỏi gì cả.

4

Compact thủ công ở ~50% context

Gõ /compact trước khi context đầy. Claude hoạt động kém ở 20% cuối cùng.

5

Commit thường xuyên

Commit code hàng ngày tối thiểu. Claude có thể undo bằng git nếu bạn commit thường xuyên. Không commit = mất an toàn.

⚠️
Cảnh báo: "Vùng ngu ngốc" (Dumb Zone)

Có một ngưỡng nguy hiểm: nếu bạn cấu hình Claude Code quá ít, nó hoạt động ổn nhưng hạn chế. Nếu cấu hình vừa đủ phức tạp nhưng chưa đúng cách, nó có thể hoạt động tệ hơn mặc định! Hãy bắt đầu với "vanilla Claude Code" (không cấu hình gì) và thêm dần từng tính năng một.

Chọn model AI phù hợp

Dự án này minh hoạ cách phân công model dựa trên độ phức tạp — giống như công ty dùng nhân viên cấp khác nhau cho việc khác nhau:

🐇
Haiku — Thực tập sinh
Nhanh, rẻ, đủ cho việc đơn giản. Dùng cho commands (điều phối), worker agents, pair programming.
🏃
Sonnet — Kỹ sư chính
Cân bằng sức mạnh và tốc độ. Dùng cho phát triển chính, agents cần suy luận, điều phối multi-agent.
🧠
Opus — Kiến trúc sư
Suy luận sâu nhất. Dùng cho quyết định kiến trúc, phân tích phức tạp, nghiên cứu.

Áp dụng cho dự án của bạn

1

Bắt đầu với CLAUDE.md

Viết sổ tay dự án: tech stack, cách build, cách test, quy ước đặt tên. Dưới 200 dòng.

2

Thêm settings.json

Cấu hình quyền hạn cơ bản: cho phép Edit, hỏi trước khi git push, cấm xoá file quan trọng.

3

Tạo Commands cho workflow lặp lại

Ví dụ: /review-pr, /deploy, /generate-tests — mỗi workflow phổ biến một command.

4

Tạo Agents chuyên biệt khi cần

Khi một task cần context riêng + tools riêng + kiến thức riêng → tạo agent. Nạp skills phù hợp.

5

Thêm Hooks khi đã ổn định

Auto-format, type-checking, security alerts. Thêm từng hook một, kiểm tra kỹ trước khi thêm hook tiếp.

🎓
Tổng kết: Từ chatbot đến đội ngũ AI

Bạn vừa học cách biến Claude Code từ một chatbot đơn lẻ thành cả một đội ngũ AI chuyên biệt. Commands điều phối, Agents xử lý, Skills cung cấp kiến thức, Hooks tự động hoá, CLAUDE.md giữ bộ nhớ, và Settings kiểm soát tất cả. Dự án claude-code-best-practice là bản thiết kế hoàn chỉnh — hãy clone nó và bắt đầu tuỳ biến cho dự án của bạn!
